# Aegis AI - Autonomous Security Orchestrator

![Aegis AI Cover](https://github.com/user-attachments/assets/cover-image.png)

Aegis AI is an advanced, autonomous Red Team platform designed to eliminate the noise of traditional vulnerability scanners. Built for the SpeedRun Hackathon, Aegis AI utilizes a specialized swarm of AI agents to not only discover vulnerabilities but to logically chain them together to identify critical paths to compromise.

## Core Architecture

Aegis AI moves beyond standard CVSS scoring. It utilizes a **Model Council** architecture:

1. **Agent Swarm Deployment:** Specialized agents (Network, Static Analysis, Dynamic Web) are deployed against the target concurrently.
2. **Telemetry Aggregation:** Agents feed their raw findings back to the central orchestration engine.
3. **Exploit Chaining (Model Council):** A powerful LLM evaluates the raw telemetry. Instead of returning 50 uncontextualized low-severity alerts, the Model Council cross-references the findings. It identifies if a low-severity bug discovered by Agent A can be combined with a medium-severity bug discovered by Agent B to create a **CRITICAL** exploit chain (e.g., Leaked RSA Key + Open SSH Port = RCE).
4. **Remediation Protocol:** Aegis outputs a clean, prioritized intelligence report containing only verified chains and actionable remediation steps.

## Tech Stack

This project was built as a lightweight, deployable prototype MVP.

* **Frontend:** React, Vite, TailwindCSS, Framer Motion
* **Backend:** Python (FastAPI), Langchain, OpenAI/OpenRouter (Llama 3.1)
* **Scanning Engine:** OWASP ZAP (Dockerized API integration), Simulated Agents
* **Target Environment:** OWASP Juice Shop (Dockerized)

## Local Deployment Setup

The application is container-ready to ensure a reproducible "0-problem" deployment environment.

### Prerequisites
* Docker & Docker Compose
* Node.js (v18+)
* Python 3.10+

### 1. Setup Backend & Services
The backend orchestrates the AI and the scanning containers.

```bash
cd backend

# Create virtual environment and install dependencies
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Start the Scanning Targets and Tools (Juice Shop & OWASP ZAP)
docker-compose up -d

# Start the FastAPI Orchestrator
uvicorn main:app --reload
```

*Ensure you have an `.env` file in the `backend` directory containing your `OPENROUTER_API_KEY` for the Model Council.*

### 2. Setup Frontend
The frontend dashboard provides the command interface for the Aegis swarm.

```bash
cd frontend

# Install dependencies
npm install

# Start the Vite dev server
npm run dev
```

Visit `http://localhost:5173` to access the Aegis AI dashboard.

## Usage Guide

1. Navigate to the Configuration Dashboard.
2. Enter the Target Domain. If running locally with the provided Docker Compose file, use `http://juice-shop:3000` to target the local OWASP Juice Shop container.
3. Select your Attack Vectors.
4. Click **Deploy Autonomous Swarm**.
5. Watch the Live Telemetry as the agents scan the target through the ZAP API.
6. Review the finalized **Security Intelligence Report** generated by the Model Council.

## Hackathon Demo Notes

* The `DynamicWebAgent` performs real, live Passive Spidering and vulnerability scanning against the Juice Shop container using the OWASP ZAP API.
* The `NetworkAgent` and `StaticAnalysisAgent` utilize hardcoded mock data to consistently demonstrate the Model Council's core value proposition: **Exploit Chaining** (e.g., taking an RSA key and an open port to deduce RCE).
